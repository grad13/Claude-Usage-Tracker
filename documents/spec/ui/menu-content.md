## Overall Menu Structure

`MenuContent` is the dropdown menu body of `MenuBarExtra`. It observes `UsageViewModel` via `@ObservedObject` and reactively updates all items.

## Usage Display Format

| Item | Format | Display Condition | Resets Text Condition |
|------|--------|-------------------|----------------------|
| 5-hour | `5-hour: XX.X%` or `5-hour: XX.X%  (resets in Xh Xm)` | `fiveHourPercent` is non-nil | Appended when `fiveHourRemainingText` is non-nil (same line, not an additional row) |
| 7-day | `7-day: XX.X%` or `7-day: XX.X%  (resets in Xd Xh)` | `sevenDayPercent` is non-nil | Appended when `sevenDayRemainingText` is non-nil (same line) |

- One decimal place (`%.1f` specifier)
- When the percentage value is nil, the entire row is hidden (no `--` placeholder is shown)
- Reset time text is generated by `DisplayHelpers.remainingText(until:)`

## Error Display Behavior

- `Text("Error: <message>")` is shown only when `viewModel.error` is non-nil
- Displayed in red via `.foregroundColor(.red)`
- Placed directly below the usage text and above the first Divider
- Error and usage text can appear simultaneously (the error does not hide usage values)

## Sign In / Sign Out Toggle Logic

Switches exclusively based on `viewModel.isLoggedIn`.

| isLoggedIn | Display | Action |
|-----------|---------|--------|
| `true` | "Sign Out" button | Calls `viewModel.signOut()` |
| `false` | "Sign In..." button (red text) | Opens the login window via `openWindow(id: "login")` and brings the app to the foreground with `NSApp.activate(ignoringOtherApps: true)` |

- The "Sign In..." text is displayed in red via `.foregroundColor(.red)`
- The trailing `...` follows the macOS convention indicating a modal operation (a separate window will open)

## Refresh Button Disabled State

```swift
Button("Refresh") { viewModel.fetch() }
    .disabled(viewModel.isFetching)
```

- While `isFetching == true`, the button is disabled (grayed out, not clickable)
- Triggers a manual fetch via `viewModel.fetch()`
- Double-fetch prevention: `fetch()` also has an internal `guard !isFetching` check

## Analysis Button Behavior

```swift
Button("Analysis") {
    openWindow(id: "analysis")
    NSApp.activate(ignoringOtherApps: true)
}
```

- Opens the Analysis window (`AnalysisWindowView`) via `openWindow(id: "analysis")`
- Brings the app to the foreground with `NSApp.activate(ignoringOtherApps: true)`
- Window ID: `"analysis"` (corresponds to the Scene definition in `ClaudeUsageTrackerApp.swift`)

## Usage Page (Browser) Button Behavior

```swift
Button("Usage Page (Browser)") {
    if let url = URL(string: "https://claude.ai/settings/usage") {
        NSWorkspace.shared.open(url)
    }
}
```

- Opens `https://claude.ai/settings/usage` in the default browser
- Uses `NSWorkspace.shared.open()` (navigates outside the app)
- If URL creation fails, nothing happens (practically never occurs)

## Version Display

The version string is displayed between the last Divider and the Quit button.

| Property | Value |
|----------|-------|
| Format | `v{CFBundleShortVersionString}` (e.g. `v0.3.0`). Falls back to `v?` if the key is missing |
| Font | `.footnote` |
| Color | `.secondary` (via `.foregroundStyle(.secondary)`) |
| Interactivity | Static `Text` -- not a button, not selectable |
